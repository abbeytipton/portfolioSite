Scrum Flow

Scrum Flow is a digital Scrum Board and organizing tool for development teams. Issues are organized into Projects and Epics and presented on the board as Stories (green header), Bugs (red header), or Defects (yellow header), organized into columns based on where the issue is in the process. Items are moved from one column to another by dragging and dropping them.

Note: Click any image to see it larger. User the right and left arrows to navigate to the next page.

Scrum Flow - Issue Details

When a user chooses to view the details of an Issue, they will see a summary of all of the information pertaining to it, including comments, which are added automatically whenever an issue is created, edited, or moved.

Any attachments that have been uploaded for the issue will be available for download on this page. All users can download an export (to PDF) of the information about the issue. Admin users will see the option to edit the issue.

Scrum Flow - Updating Progress

As developers work on their issues, they use this page to update their progress. Users slide the bars to the right to increase the amount of hours they're logging and how much of the issue is finished. They can also leave a comment and upload any documentation related to the issue.

Scrum Flow - Test Cases

After finishing an issue, developers use this page to add a Test Case. They choose a code reviewer and a tester, and then type detailed information about the purpose of the issue, how it was solved, and what the reviewer and tester should be looking for.

The text boxes on this page allow formatting, but do not allow extraneous code, copy/pasting of spaces, or other such skips. Each box also has a character requirement of 200 characters.

Scrum Flow - Suggesting Tasks

Regular users can suggest new tasks that they would like to work on. These task suggestions are then sent to admin users, who review the suggestions and determine if they should be added or not. Users might need to suggest tasks as they work on an issue and discover a need, or if they have found something in their work that should be redone.

Scrum Flow - Approving Tasks

When an admin user reviews task suggestions, they see a table of each suggestion. Suggestions can come from users of this web application, or from an accompanying application that lets customers report problems and make suggestions. Admin users can use the buttons in each row to add the suggestion to the Current Sprint, the Backlog, add it as an Epic (if it is a broad topic rather than a single issue), or Reject the suggestion.

If the user chooses to reject a suggestion, they must give a reason. After any selection on this page, a second admin must agree to the selections before the item will be added or rejected.

Scrum Flow - Bug Reports

Admin users can also see bug reports from customers. Customer enter these reports on an accompanying web application. Admin users get a notification by email when a customer reports a new bug, and they can then view their bugs here. The bugs can be added to the Current Sprint, the Backlog, or Rejected.

After a second admin has approved the selection, the admin user will get the opportunity to email the customer about their report. The email can tell the customer when they should expect a fix, and can request any further information or data needed.

Scrum Flow - Admin Functions

The layout has two different menus; in this side navigation menu, the admin user will see a few more options that a regular user. They have an expandable section (seen here expanded) that links them to all the admin or manager related pages. Regular users cannot access these pages, even if they attempt to go to them directly.

Scrum Flow - User Menu

This dropdown menu is the same for all users and lets the user customize the look and feel of the web application. The icon next to the user's username is chosen by them, including the color. When a user has unread alerts, a bubble will appear over the icon showing how many alerts they have.

Scrum Flow - Updating Account

On the Update Account page, users can change their basic information, as well as update their profile picture. The user can choose from over 20 icons and 12 colors to customize the look of the page to their liking. The icons are generated from Font Awesome, and admin users can change these choices quite easily.

Scrum Flow - Updating Settings

On the Settings page, users can change how pages are viewed. They can change their mode to Light, Cool, or Dark - they can choose to preview the change or save the changes, which will be remembered the next time they log in.

Users can also change their default filters on the Scrum Board; if they'd like to default to see only their issues, only issues for a certain project, only certain issues types, or a combination, they can use the dropdown boxes here to do so.

Finally, users can update their alert settings. They can choose when they'd like to receive email alerts and alerts on the site.

Scrum Flow - Cool Mode

This is the Scrum Board in Cool Mode. The colors are more calm, while still being bright and pleasing to look at. If users choose this mode, it will be remembered the next time they visit the web application.

Scrum Flow - Dark Mode

This is the Scrum Board in Dark Mode. The colors are all darker and easier to read for long-term typing or reading. If users choose this mode, it will be remembered the next time they visit the web application.

Scrum Flow - Graphs

Admin users can access the Graphs page, which gives them a rundown of the progress the team is making in the current Sprint and in past Sprints. This view shows the Developer Progress graph - the bars are showing points assigned to the user in the current sprint vs points completed.

Other available graphs are: Quality Assurance Progress, Sprint Burndown, and Project Progress. The information can be viewed for the current sprint, all other sprints completed, or all time.

The graphs can be downloaded as images, or in Excel format by using the buttons next to the filter dropdowns.
